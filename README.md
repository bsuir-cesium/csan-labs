# CSaN Labs

Лабораторные работы по дисциплине «Сети и коммуникации» (КСиС).

## Lab 1 — Сканер локальной сети

**Задание:** создать программу, сканирующую локальную сеть и отображающую информацию об узлах
(IP, MAC-адрес, имя хоста), включая собственный компьютер.

**Реализация:**

- Двухфазное сканирование: ICMP ping каждого хоста → чтение ARP-таблицы для получения MAC и hostname
- Многопоточность через `ThreadPoolExecutor` (до 100 воркеров) — работает, потому что это
  IO-bound задача и GIL здесь не мешает
- Поддержка нескольких сетевых интерфейсов — парсинг `ifconfig`, фильтрация служебных
  интерфейсов (loopback, AirDrop, bridge и т.д.), сканирование каждой сети отдельно
- Диапазон адресов вычисляется из IP и маски подсети через `ipaddress.IPv4Network`

## Lab 2 — P2P-мессенджер (Socket API)

**Задание:** обмен текстовыми сообщениями и бинарными файлами между двумя и более
компьютерами через Socket API. WebSocket запрещён.

**Реализация:**

- Peer-to-peer архитектура: каждый узел одновременно сервер и клиент (TCP)
- Кастомный бинарный протокол: заголовок 5 байт (1 байт тип + 4 байта длина, big-endian),
  три типа сообщений — `TEXT`, `FILE`, `NICKNAME`
- IP и порт не захардкожены — порт вводится при запуске, подключение к пирам через
  `/connect <host> <port>`
- Проверка доступности порта при старте сервера — если `bind()` бросает `OSError`,
  приложение завершается с ошибкой
- Потокобезопасное управление списком пиров (`threading.Lock`)
- Автоматическое разрешение коллизий имён файлов при приёме (суффикс `_1`, `_2`, ...)
